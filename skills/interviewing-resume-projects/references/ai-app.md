# AI 应用项目探查参考

仅在简历陈述或候选人回答提供相应信号后使用这些维度。示例关键词、风险和变式都不是项目事实，必须继续索取可归属、可复现的证据。

### 模型选择
- 简历信号：模型选型、模型对比、开源/闭源模型、微调、路由
- 核验贡献：要求说明候选模型、任务约束、基线、评测结果及本人做出的选择
- 深化：质量、延迟、成本、上下文、部署与锁定效应取舍
- 变式：模型版本、流量结构或合规约束变化时如何重新选型

### Prompt 生命周期
- 简历信号：Prompt 工程、模板、版本管理、提示词优化、A/B 测试
- 核验贡献：要求展示从问题样例到提示词变更、评测、发布和回退的本人操作证据
- 深化：变量边界、版本、回归、模型耦合与可维护性
- 变式：模型升级或用户输入分布漂移时如何发现提示词退化

### 结构化输出
- 简历信号：JSON、Schema、function calling、解析器、格式遵循率
- 核验贡献：要求说明输出契约、校验/修复路径及非法输出样本与指标
- 深化：严格 Schema、语义校验、流式解析、兼容与拒绝策略
- 变式：字段缺失、类型正确但语义错误或 Schema 升级时如何处理

### RAG 摄取
- 简历信号：知识库、文档解析、切分、向量化、索引构建、增量更新
- 核验贡献：要求还原数据来源、清洗切分、元数据、更新链路及本人验证的数据质量
- 深化：切块粒度、重复、版本、权限、解析失败与索引新鲜度
- 变式：文档格式复杂、频繁更新或同一事实多版本并存时如何摄取

### RAG 检索
- 简历信号：向量检索、混合检索、BM25、召回率、查询改写
- 核验贡献：要求说明查询、候选集、过滤、召回证据及本人调过的关键参数
- 深化：稀疏/稠密、top-k、元数据过滤、语义漂移与召回-精度取舍
- 变式：专有名词、短查询或权限过滤导致零召回时如何降级

### RAG 重排
- 简历信号：rerank、交叉编码器、二阶段检索、相关性排序
- 核验贡献：要求说明重排输入、模型/规则、截断位置及相对无重排基线的证据
- 深化：候选深度、位置偏差、延迟成本、相关性标注和排序稳定性
- 变式：候选量激增或重排服务超时时如何维持可接受结果

### 事实落地与引用
- 简历信号：grounding、引用、出处、可追溯回答、原文定位
- 核验贡献：要求展示答案主张到来源片段/版本的映射及引用正确性检查
- 深化：支持性与相关性、引用粒度、冲突来源、无证据拒答边界
- 变式：检索到相互冲突或过期来源时如何呈现和决策

### 评估数据集与指标
- 简历信号：评测集、准确率、召回率、faithfulness、人工评审、benchmark
- 核验贡献：要求说明样本来源、标注、切分、指标定义、基线和本人误差分析
- 深化：代表性、泄漏、主观指标一致性、置信区间与分层结果
- 变式：离线指标提升但真实用户满意度下降时如何修订评测

### 幻觉
- 简历信号：幻觉治理、事实性、拒答、置信度、可信回答
- 核验贡献：要求给出幻觉定义、失败样本、检测方法及措施前后的可比较证据
- 深化：知识缺失与推理错误、过度拒答、校准及无法完全消除的边界
- 变式：高风险领域或来源不足时如何调整回答与升级路径

### 延迟
- 简历信号：响应优化、首 token、P95/P99、流式输出、加速
- 核验贡献：要求拆解端到端延迟、测量环境、瓶颈及本人优化前后数据
- 深化：排队、检索、推理、网络、首 token 与总时长取舍
- 变式：并发突增或外部模型变慢时如何守住延迟预算

### Token 成本
- 简历信号：token 优化、成本降低、上下文压缩、缓存、模型路由
- 核验贡献：要求说明计费口径、单请求/任务成本基线及本人改动带来的质量-成本变化
- 深化：输入/输出 token、隐藏重试、缓存命中、长上下文边际收益
- 变式：价格或输入长度分布变化时如何重算并设置预算

### 降级与兜底
- 简历信号：fallback、降级、备用模型、规则兜底、熔断
- 核验贡献：要求说明触发条件、替代路径、用户可见结果及演练或线上证据
- 深化：质量下限、超时预算、熔断恢复、语义兼容与失败透明度
- 变式：主备同时不可用或备用模型输出契约不一致时如何处理

### 隐私
- 简历信号：隐私、PII、脱敏、数据隔离、合规、本地部署
- 核验贡献：要求说明数据流、保留策略、访问控制、脱敏点及本人验证的控制措施
- 深化：最小化采集、第三方传输、日志泄露、删除、租户隔离边界
- 变式：用户请求删除或供应商保留政策变化时如何端到端落实

### 追踪
- 简历信号：LLM tracing、调用链、Prompt/响应日志、token/延迟监控、调试平台
- 核验贡献：要求展示一次从请求到检索、模型、解析的关联追踪及其定位作用
- 深化：trace/span、版本标签、脱敏、采样、成本和质量关联
- 变式：仅少量用户出现偶发错误时如何保留足够且合规的证据

### 生产反馈闭环
- 简历信号：用户反馈、线上评测、A/B、数据飞轮、持续优化、回归监控
- 核验贡献：要求说明反馈采集、归因、样本回流、变更验证及本人推动的一次闭环
- 深化：选择偏差、显隐式反馈、延迟标签、污染、发布门槛与回滚
- 变式：反馈稀疏、被刷或与离线结论冲突时如何校正
