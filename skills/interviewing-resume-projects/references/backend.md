# 后端项目探查参考

仅当简历陈述或候选人回答出现对应信号时，才选用下列维度。关键词和追问方向只是触发器，不代表项目事实；所有结论仍需候选人证据支持。

### 数据库表结构与索引
- 简历信号：表设计、数据建模、索引优化、慢查询、分库分表
- 核验贡献：要求说明亲自设计或修改的表、索引及其依据，并用执行计划或前后指标归属效果
- 深化：字段与约束、联合索引顺序、选择性、覆盖索引及写放大取舍
- 变式：数据量或读写比例显著变化时，原设计如何失效或演进

### SQL 证据
- 简历信号：SQL 优化、查询提速、报表计算、复杂联表
- 核验贡献：要求还原关键 SQL 的结构、输入规模、执行计划与可复现的优化前后证据
- 深化：连接顺序、基数估计、回表、排序、锁与统计信息边界
- 变式：数据倾斜或参数分布变化后，计划稳定性与修正方式

### 缓存一致性与失效
- 简历信号：Redis、缓存加速、多级缓存、热点数据、命中率
- 核验贡献：要求说明缓存读写路径、失效策略、异常处置及本人实现和验证的部分
- 深化：一致性窗口、穿透/击穿/雪崩、脏读、热键与缓存更新取舍
- 变式：缓存不可用、延迟升高或发生批量过期时的系统行为

### 并发与事务
- 简历信号：高并发、事务、锁、幂等、库存/余额/订单一致性
- 核验贡献：要求给出并发冲突场景、事务边界、隔离或锁策略，以及竞态测试或故障证据
- 深化：隔离级别、死锁、乐观/悲观控制、原子性与跨服务一致性边界
- 变式：并发量增大、请求重放或部分提交时如何保持业务不变量

### API 契约
- 简历信号：接口设计、REST/RPC、开放平台、服务拆分、版本升级
- 核验贡献：要求展示本人定义的请求响应、错误语义、兼容策略及调用方验证证据
- 深化：幂等键、分页、超时、版本化、鉴权和契约演进取舍
- 变式：下游重复调用、字段缺失或新旧客户端并存时如何处理

### 可观测性
- 简历信号：监控、日志、链路追踪、告警、故障定位、SLA/SLO
- 核验贡献：要求说明亲自埋设的指标/日志/追踪点，以及它们如何缩小一次真实问题的定位范围
- 深化：黄金信号、关联标识、采样、告警阈值与可行动性边界
- 变式：跨服务间歇性故障或监控自身缺失时如何补证据

### 负载与容量
- 简历信号：压测、吞吐、QPS/TPS、延迟、容量规划、性能提升
- 核验贡献：要求说明负载模型、环境、瓶颈证据、基线和本人推动的容量结论
- 深化：吞吐与尾延迟、排队、资源饱和、流控及压测代表性
- 变式：流量突增、热点迁移或依赖容量下降时如何重估

### 部署
- 简历信号：CI/CD、容器、Kubernetes、灰度发布、自动化部署
- 核验贡献：要求还原本人负责的制品、配置、发布步骤、健康检查及发布验证
- 深化：配置隔离、就绪/存活检查、滚动与灰度策略、数据变更顺序
- 变式：发布中实例异常、迁移未完成或新旧版本同时运行时如何处置

### 回滚
- 简历信号：回滚、故障恢复、发布事故、灾备、版本治理
- 核验贡献：要求说明可执行的回滚触发条件、步骤、耗时与一次演练或事故证据
- 深化：代码、配置、数据库和消息格式的可逆性及恢复点取舍
- 变式：不可逆数据迁移或回滚后依赖仍不兼容时采用何种前滚/补偿方案
